Automation Framework Structure

project_name
/data
Stores test data (e.g., user credentials, URLs)
Example: test_data.json: {"username": "test_user", "password": "securePass123"}
/features
Contains Gherkin feature files describing application behavior
login.feature
Scenario: Validate user login functionality
/steps
Maps Gherkin steps to Python code
login_steps.py
Implements steps for "Given", "When", "Then"
/utilities
Reusable modules (e.g., logging, database helpers)
logger.py
Provides centralized logging for debugging
db_helper.py
Handles CRUD operations on the database
/reports
Stores test execution reports
report.html
HTML report summarizing test outcomes
environment.py
Configures setup/teardown (e.g., browser, DB)
behave.ini
Runtime settings (e.g., browser type, test tags)
requirements.txt
Specifies project dependencies (e.g., Selenium, Behave)
